QACache-Micro 实现手册

简介

QACache-Micro 是 QACache 的轻量级版本，专为个人用户和小团队设计，旨在提供快速响应和高召回率的缓存体验。它具有以下特点：

  • 轻量化：内存占用 50MB-200MB（根据模式自适应）
  • 快速部署：一键启动，无需复杂配置，支持 qacache -s start
  • 高召回率：双引擎架构，兼顾精确匹配和语义搜索
  • 离线支持：可在无网络环境下工作
  • 自适应：根据设备性能自动选择最优搜索引擎

核心设计理念

自适应双引擎架构

模式选择策略：

  • 轻量模式：内存 < 1GB 或用户指定，使用 SQLite FTS5 进行文本搜索
  • 标准模式：内存 >= 1GB 且支持向量计算，使用 Faiss + BGE 进行语义搜索
  • 混合模式：两种引擎并存，结果融合排序

响应优先级

轻量模式：

  1. 内存缓存：毫秒级响应，存储最近访问的热点数据
  2. FTS5 搜索：十毫秒级响应，基于关键词和短语匹配
  3. LLM 调用：秒级响应，作为最后的兜底方案

标准模式：

  1. 内存缓存：毫秒级响应，存储最近访问的热点数据
  2. 向量搜索：十毫秒级响应，基于语义相似度匹配
  3. FTS5 回退：百毫秒级响应，向量搜索失败时的备选方案
  4. LLM 调用：秒级响应，作为最后的兜底方案

技术选型

  • Web 框架：FastAPI（高性能异步）
  • 存储引擎：SQLite（轻量级本地数据库）
  • 文本搜索：SQLite FTS5（内置全文搜索，支持中文分词）
  • 向量搜索：Faiss（CPU 优化，可选 GPU 加速）
  • 嵌入模型：BGE-small-zh-v1.5（384 维，快速推理）
  • 部署方式：Python 包 + 系统服务（真正的一键部署）

性能优化策略

自适应优化

  • 动态模式切换：根据查询模式和系统负载自动在轻量/标准模式间切换
  • 智能缓存策略：基于访问频率和查询相似度优化内存缓存
  • 渐进式索引：后台异步构建向量索引，不影响服务启动速度

资源管理

  • 内存监控：实时监控内存使用，超限时自动降级到轻量模式
  • 磁盘清理：定期清理过期缓存和无效索引
  • 模型懒加载：按需加载嵌入模型，减少启动时间

查询优化

  • 结果融合：FTS5 和向量搜索结果的智能融合排序
  • 查询重写：自动优化用户查询，提高匹配准确率
  • 缓存预热：根据历史查询模式预加载热点数据

发展路线图

短期目标（1-2个月）

  • ✅ 实现双引擎架构（FTS5 + Faiss）
  • ✅ 完成一键部署系统
  • 🔄 添加 Web 管理界面
  • 🔄 实现查询结果融合算法

中期目标（3-6个月）

  • 📋 支持多种嵌入模型（BGE、M3E、text2vec）
  • 📋 实现分布式部署支持
  • 📋 添加 RESTful API 和 SDK
  • 📋 集成更多数据源（文档、网页、数据库）

长期目标（6个月+）

  • 📋 实现用户行为分析和个性化推荐
  • 📋 支持多语言和跨语言搜索
  • 📋 集成大语言模型进行智能问答
  • 📋 开发企业版功能（权限管理、审计日志）

技术演进

  • 模型更新：支持模型热更新，无需重启服务
  • 硬件适配：支持 GPU 加速、ARM 架构
  • 云原生：支持 Kubernetes 部署和自动扩缩容
  • 生态集成：与主流开发框架和工具链集成
