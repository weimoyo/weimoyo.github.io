QACache 三层架构设计文档

项目概述

QACache 是一个基于语义缓存的大模型加速系统，采用三层分布式架构设计，旨在通过缓存相似查询的响应来减少大模型调用次数，提升响应速度并降低服务器负载。

整体架构设计

三层缓存架构

架构特点

  • 用户侧：轻量化、实时交互、高召回率、低精度
  • CDN网关：数据收集、智能分发、模式分析
  • 服务器侧：高性能、精准匹配、低召回率、不增加网络负载

1. 用户侧轻量缓存（Client-Side Cache）

设计目标

  • 一键部署，避免繁琐配置
  • 内存占用小，响应速度快
  • 高召回率，为用户提供充分参考
  • 支持离线工作模式

技术选型

核心组件

  • 存储引擎：SQLite（轻量级本地数据库）
  • 关键词检索：FTS5（基于关键词的传统检索）
  • 向量搜索：Faiss（轻量级向量搜索）
  • 嵌入模型：BGE-small-zh-v1.5（小模型，快速推理）
  • Web框架：FastAPI（高性能异步框架）

设计目标

  • 收集用户查询模式和习惯
  • 智能分发缓存数据
  • 提供冷启动数据支持
  • 负载均衡和就近访问

3. 服务器侧重型缓存（Server-Side Cache）

设计目标

  • 高性能、精准匹配
  • 低召回率，避免网络负载
  • 支持大规模数据存储
  • 提供高质量缓存源

技术选型

  • 搜索引擎：Elasticsearch（支持分片和集群）
  • 向量模型：BGE-base-zh-v1.5（高精度嵌入）
  • 缓存策略：LRU + TTL + 质量评分

架构优势

  1. 用户体验：客户端快速响应，减少等待时间
  2. 资源优化：按需分配计算资源，避免浪费
  3. 可扩展性：支持水平扩展，适应不同规模需求
  4. 数据智能：通过CDN收集数据，优化缓存策略
  5. 部署灵活：支持多种部署模式，满足不同场景
